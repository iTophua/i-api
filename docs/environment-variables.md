# 环境变量使用指南

## 概述

环境变量是 API 测试中非常重要的功能，它允许你在不同的环境下使用不同的配置。常见的场景包括：
- 开发环境、测试环境、生产环境的 API 地址切换
- 敏感信息（如 API Key、Token）的统一管理
- 请求参数的可变部分（如用户 ID、版本号）

## 核心概念

### 1. 环境（Environment）

环境是一组变量的集合。每个环境包含：
- **环境名称**：用于识别环境（如 "开发环境"、"测试环境"）
- **变量列表**：键值对集合，支持启用/禁用状态

### 2. 变量（Variable）

变量由三部分组成：
- **变量名（Key）**：引用的标识符
- **变量值（Value）**：实际的值
- **启用状态**：控制该变量是否生效

## 使用方法

### 创建环境

1. 点击左侧导航栏中的 **环境管理** 图标
2. 点击右侧顶部的 **"+"** 按钮
3. 输入环境名称，点击 **创建**

### 添加变量

1. 在环境详情面板中，点击 **"添加变量"** 按钮
2. 填写变量名和变量值
3. 启用/禁用开关控制变量是否生效

### 编辑变量

直接在变量表格中修改变量名或变量值，修改会自动保存。

### 删除变量

点击变量行右侧的 **删除** 按钮（垃圾桶图标）可删除该变量。

### 删除环境

1. 选中要删除的环境
2. 点击环境名称右侧的 **删除环境** 按钮
3. 在确认对话框中点击 **确定**

> ⚠️ 注意：默认环境（default）无法删除

### 切换当前环境

在环境管理面板中，点击左侧列表中的环境即可切换为当前环境。

## 变量引用方式

在请求中使用环境变量需要使用特定的语法：

### 语法格式

```
{{变量名}}
```

### 使用示例

假设你创建了一个名为 `dev` 的环境，包含以下变量：

| 变量名 | 变量值 |
|--------|--------|
| baseUrl | https://api.example.com |
| apiKey | abc123 |
| version | v2 |

#### 在 URL 中使用

```
{{baseUrl}}/users
```

发送请求时，`{{baseUrl}}` 会被替换为 `https://api.example.com`，实际 URL 变为：
```
https://api.example.com/users
```

#### 在 Headers 中使用

```json
{
  "Authorization": "Bearer {{apiKey}}",
  "X-API-Version": "{{version}}"
}
```

#### 在请求体中使用

```json
{
  "name": "test",
  "endpoint": "{{baseUrl}}/resource"
}
```

## 作用域与优先级

### 变量作用域

每个变量仅在其所属的环境内有效。当切换环境时，使用的变量集合也会相应改变。

### 变量优先级

- 当多个环境中存在相同名称的变量时，**当前激活的环境**中的变量值优先
- 未启用的变量不会被解析，即使存在同名变量

## 最佳实践

### 1. 命名规范

- 变量名建议使用驼峰命名法或下划线分隔
- 保持命名的一致性，便于记忆和引用

### 2. 环境分组

- 按环境类型分组（开发、测试、生产）
- 按功能模块分组（认证、API、配置）

### 3. 敏感信息

- API Key、Token 等敏感信息不要明文写在请求中
- 使用环境变量统一管理，启用版本控制时忽略敏感值

### 4. 变量注释

- 为重要变量添加描述说明用途
- 在变量名中体现其用途（如 `authToken` 而非 `token`）

## 常见问题

### Q: 为什么变量没有生效？

1. 检查变量是否处于**启用**状态
2. 确认当前环境是否正确
3. 检查变量名拼写是否与引用一致（区分大小写）

### Q: 如何在不同环境间快速切换？

在环境管理面板中，点击左侧环境列表中的目标环境即可切换。

### Q: 默认环境有什么特殊之处？

默认环境（default）是系统自动创建的环境，包含基础的变量配置。它无法被删除，但可以重命名和编辑其中的变量。

### Q: 变量支持哪些字符？

变量名支持：
- 英文字母（a-z, A-Z）
- 数字（0-9）
- 下划线（_）
- 变量名不能以数字开头
